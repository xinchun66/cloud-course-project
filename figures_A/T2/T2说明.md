# T2 CCE 集群搭建说明

## 完成情况

T2 已完成。当前华为云 CCE 集群已创建并配置 2 个 Worker 节点，kubectl 可以正常连接集群。

关键配置如下：

| 项目 | 内容 |
|---|---|
| 区域 | 华北-北京四 |
| 集群名称 | cloud-course-cce |
| 集群类型 | CCE Standard |
| 计费模式 | 按需计费 |
| Kubernetes 版本 | v1.34 |
| Worker 节点数量 | 2 |
| Worker 节点规格 | c9.large.2，2 vCPUs / 4 GiB |
| 操作系统 | Ubuntu 22.04.5 LTS |
| 容器运行时 | containerd |
| VPC | cloud-course-vpc，192.168.0.0/16 |
| 子网 | cloud-course-subnet，192.168.0.0/24 |

## 报告可用说明

本任务在华为云北京四区域创建了 CCE Standard 集群 `cloud-course-cce`，采用按需计费模式。集群版本为 v1.34，满足任务书中 Kubernetes 版本不低于 1.27 的要求。随后创建 2 个 2 vCPU / 4 GiB 的 Worker 节点，节点操作系统为 Ubuntu 22.04.5 LTS，容器运行时为 containerd。通过 CloudShell 执行 `kubectl get nodes -o wide`，两个 Worker 节点均处于 Ready 状态，输出中包含 VERSION 列，说明 CCE 集群和 kubectl 配置均已完成。
