# T3 应用部署说明

## 完成状态

T3 已完成。当前已将 Flask 后端、Redis 和 Nginx 前端部署到华为云 CCE 集群中。后端 Deployment 副本数为 2，Redis 副本数为 1，前端副本数为 1。后端 Service 通过华为云 ELB 对外暴露，公网访问 `/api/ping` 返回正常。

## 镜像与资源配置

| 资源 | 名称 | 说明 |
|---|---|---|
| Backend Deployment | `backend` | 副本数 2，镜像为 `swr.cn-north-4.myhuaweicloud.com/cloud-course-a/backend:v1`。 |
| Frontend Deployment | `frontend` | 副本数 1，镜像为 `swr.cn-north-4.myhuaweicloud.com/cloud-course-a/frontend:v1`。 |
| Redis Deployment | `redis` | 副本数 1，镜像为 `redis:7-alpine`。 |
| Backend Service | `backend-svc` | `LoadBalancer` 类型，对外暴露 80 端口并转发到后端 5000 端口。 |
| Redis Service | `redis-svc` | `ClusterIP` 类型，仅集群内部访问。 |
| Frontend Service | `frontend-svc` | `ClusterIP` 类型，仅集群内部访问。 |
| ConfigMap | `backend-config` | 注入 Redis 地址、端口和运行环境。 |
| Secret | `redis-secret` | 注入 Redis 密码字段。 |

由于镜像仓库位于成员 A 的 SWR 组织 `cloud-course-a`，后端和前端 Deployment 使用 CCE 自动创建的 `default-secret` 作为 `imagePullSecrets`，用于拉取私有 SWR 镜像。

## 执行命令

部署资源：

```bash
kubectl apply -f k8s/redis-pvc.yaml
kubectl apply -f k8s/backend-config.yaml
kubectl apply -f k8s/redis-secret.yaml
kubectl apply -f k8s/redis-deployment.yaml
kubectl apply -f k8s/backend-deployment.yaml
kubectl apply -f k8s/services.yaml
kubectl apply -f k8s/frontend-nginx-config.yaml
kubectl apply -f k8s/frontend-deployment.yaml
```

由于后端和前端镜像为私有 SWR 镜像，部署后补充绑定镜像拉取密钥：

```bash
kubectl patch deployment backend -p '{"spec":{"template":{"spec":{"imagePullSecrets":[{"name":"default-secret"}]}}}}'
kubectl patch deployment frontend -p '{"spec":{"template":{"spec":{"imagePullSecrets":[{"name":"default-secret"}]}}}}'
```

由于 `backend-svc` 初始状态下 ELB 公网 IP 一直处于 `<pending>`，在华为云控制台手动创建共享型公网 ELB `cloud-course-backend-elb` 后，将该 ELB 绑定到 Service：

```bash
kubectl annotate svc backend-svc kubernetes.io/elb.id=e4ed811c-977a-4146-9848-705ab93f59a6 --overwrite
kubectl annotate svc backend-svc kubernetes.io/elb.class=union --overwrite
```

检查资源状态：

```bash
kubectl get pods -o wide
kubectl get deploy -o wide
kubectl get svc -o wide
kubectl get configmap,secret
```

访问接口：

```bash
curl http://1.92.151.224/api/ping
```

返回结果：

```json
{"redis":"connected","status":"ok"}
```

## 问题与解决

### 问题 1：私有 SWR 镜像拉取失败

部署后 backend 和 frontend Pod 初始出现 `ErrImagePull` / `ImagePullBackOff`。通过 `kubectl describe pod` 查看 Events，发现错误为 `401 Unauthorized`，说明 CCE 节点无法匿名拉取私有 SWR 镜像。

解决方法：确认集群中存在 `default-secret`，并将其作为 `imagePullSecrets` 绑定到 backend 和 frontend Deployment。修复后 Pod 成功拉取镜像并进入 `Running` 状态。

### 问题 2：LoadBalancer Service 一直 Pending

`backend-svc` 初始创建后 `EXTERNAL-IP` 一直为 `<pending>`。通过 `kubectl describe svc backend-svc` 查看 Events，发现 CCE 提示未指定 `kubernetes.io/elb.id` 或 `loadBalancerIP`。

解决方法：在华为云 ELB 控制台创建共享型公网 ELB，并通过 `kubectl annotate svc backend-svc kubernetes.io/elb.id=... --overwrite` 将 ELB 绑定到 Service。绑定后 `backend-svc` 获得公网 IP `1.92.151.224`，公网访问 `/api/ping` 正常。

## 结论

本任务完成了后端、Redis 和前端在 CCE 上的部署。后端通过 ConfigMap 获取 Redis 地址，通过 Secret 获取密码字段；Redis 通过 ClusterIP Service 提供集群内访问；后端通过 LoadBalancer Service 绑定华为云 ELB 暴露公网入口。最终 Pod 全部 Running，公网访问 `/api/ping` 返回 `{"redis":"connected","status":"ok"}`，说明 T3 应用部署验收通过。
