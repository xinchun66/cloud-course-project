# T4 Redis 持久化存储说明

## 完成状态

T4 已完成。当前已为 Redis 创建 PVC，并将 PVC 挂载到 Redis 容器的 `/data` 目录。随后通过写入测试数据、删除 Redis Pod、等待 Pod 重建、再次查询数据的方式，验证 Redis 数据在 Pod 重建后没有丢失。

## 资源配置

| 资源 | 名称 | 说明 |
|---|---|---|
| PVC | `redis-data-pvc` | 申请 10Gi 云硬盘存储，`storageClassName` 为 `csi-disk`。 |
| Deployment | `redis` | Redis 副本数为 1，镜像为 `redis:7-alpine`。 |
| 挂载路径 | `/data` | Redis 容器内数据目录，挂载到 `redis-data-pvc`。 |

Redis Deployment 中的挂载关系为：

```text
/data <- redis-data <- redis-data-pvc
```

## 执行命令

创建 PVC 和 Redis Deployment：

```bash
kubectl apply -f k8s/redis-pvc.yaml
kubectl apply -f k8s/redis-deployment.yaml
```

检查 PVC 绑定状态：

```bash
kubectl get pvc
```

检查 Redis Pod 运行状态：

```bash
kubectl get pods -o wide
```

查看 Redis Pod 挂载详情：

```bash
REDIS_POD=$(kubectl get pod -l app=redis -o jsonpath='{.items[0].metadata.name}')
kubectl describe pod $REDIS_POD
```

写入测试数据：

```bash
kubectl exec -it $REDIS_POD -- redis-cli SET testkey "hello"
kubectl exec -it $REDIS_POD -- redis-cli GET testkey
```

删除 Redis Pod 触发 Deployment 自动重建：

```bash
kubectl delete pod $REDIS_POD
kubectl get pods -o wide
```

重建后再次查询测试数据：

```bash
NEW_REDIS_POD=$(kubectl get pod -l app=redis -o jsonpath='{.items[0].metadata.name}')
kubectl exec -it $NEW_REDIS_POD -- redis-cli GET testkey
```

## 结论

本任务通过 PVC 将 Redis 数据目录 `/data` 持久化到华为云 EVS 云硬盘。实验中删除 Redis Pod 后，Kubernetes Deployment 自动重建了新的 Redis Pod。重建后查询 `testkey` 仍返回 `"hello"`，说明 Redis 数据没有随 Pod 删除而丢失，PVC 持久化存储配置生效。
